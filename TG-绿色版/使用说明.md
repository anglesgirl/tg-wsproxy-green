# Telegram 绿色版（内置 WSS 代理）

## 这是什么

一个零配置的 Telegram Desktop 绿色版，内置 WSS 代理隧道。

启动器自动拉起代理 → 首次运行自动配置 Telegram 代理 → 之后一键启动，关 TG 自动清理代理。

## 目录结构

```
TG-绿色版/
├── 启动Telegram.exe      ← 双击这个
├── 使用说明.md
├── .proxy_configured     ← 首次运行后自动生成（记录 secret）
├── Telegram/             ← 官方 Telegram Desktop
│   └── Telegram.exe
└── wsproxy/              ← tg-ws-proxy
    ├── TgWsProxy.exe     ← 需自行下载放入
    └── .gitkeep
```

## 使用方法

### 首次运行

1. 先下载 `TgWsProxy.exe`（从 [tg-ws-proxy Releases](https://github.com/Flowseal/tg-ws-proxy/releases) 下载），放入 `wsproxy` 目录
2. 双击 `启动Telegram.exe`
3. Telegram 自动启动，弹出"启用代理"对话框
4. 点击"启用"（或"Enable"）
5. 完成！Telegram 已通过本地代理连接

### 以后运行

直接双击 `启动Telegram.exe`，代理自动启动 → Telegram 自动连接，无需任何操作。
关闭 Telegram 后，代理自动清理。

## 工作原理

```
Telegram Desktop → MTProto Proxy (127.0.0.1:1443) → WebSocket → Telegram DC
```

1. 启动器预创建配置文件（固定 secret）到 `%APPDATA%\TgWsProxy\config.json`
2. 启动 TgWsProxy.exe（静默在系统托盘运行，不弹窗口）
3. 等待代理端口 1443 就绪
4. 首次运行：打开 `tg://proxy?server=127.0.0.1&port=1443&secret=XXX` → Telegram 自动弹出代理配置
5. 后续运行：直接启动 Telegram（代理已配好，自动连接）
6. 关闭 Telegram → 自动关闭代理

## 配置说明

### 代理配置文件

位置：`%APPDATA%\TgWsProxy\config.json`

```json
{
  "port": 1443,
  "host": "127.0.0.1",
  "secret": "ee8a1b8e0c2d4f6a9b3e7c5d1f2a4b6c",
  "fallback_cfproxy": true
}
```

- **port**: 代理端口，默认 1443
- **host**: 监听地址，默认 127.0.0.1（仅本机）
- **secret**: MTProto 密钥，固定值（确保 Telegram 配置不变）
- **fallback_cfproxy**: Cloudflare 代理回退，默认开启

### TgWsProxy 托盘菜单

代理运行后会在系统托盘显示图标，右键可操作：
- **Open in Telegram**: 和启动器一样的功能
- **Copy link**: 复制 `tg://proxy` 链接
- **Restart proxy**: 重启代理
- **Settings**: 修改端口、DC 等
- **Open logs**: 查看日志
- **Exit**: 退出代理

## 特点

- 独立 exe（Go 编译，无依赖）
- 代理静默运行在托盘，不打扰用户
- 首次运行通过 `tg://proxy` 自动配置 Telegram
- 后续运行直接启动 Telegram，自动连接代理
- 固定 secret，配置不会丢失
- 关 TG 自动清理代理进程
- 代理已运行时不重复启动

## 开机自启

右键 `启动Telegram.exe` → 创建快捷方式 → Win+R 输入 `shell:startup` → 拖进去
