# Telegram 绿色版(内置 WSS 代理)

## 这是什么

一个零编译、零修改的 Telegram Desktop 绿色启动方案。

**原理**:启动器先拉起本地 wsproxy(SOCKS5 → WSS 隧道),再启动官方原版 Telegram Desktop。
Telegram 通过 SOCKS5 代理 `127.0.0.1:1080` 连接,流量被封装成 WSS 到 `kws*.web.telegram.org`,
效果等同于安卓 Nullgram 的内置 wsproxy。

关闭 Telegram 后,启动器自动清理代理进程,不留后台。

## 目录结构

```
TG-绿色版/
├── 启动Telegram.vbs      ← 双击这个(静默,无黑窗口)
├── 启动Telegram.bat      ← 或这个(有日志,调试用)
├── Telegram/             ← 放官方 Telegram Desktop
│   └── Telegram.exe
└── wsproxy/              ← 放 tg-ws-proxy 程序
    └── TgWsProxy.exe
```

## 首次使用(三步)

### 1. 放文件

- 从 https://github.com/Flowseal/tg-ws-proxy/releases/latest 下载
  选 `TgWsProxy-windows-x64-console.zip`(控制台版,适合本方案)
  解压后把 exe 重命名为 `TgWsProxy.exe`,放到 `wsproxy\` 文件夹

- 从 https://desktop.telegram.org/ 下载 Windows 便携版
  解压后把 `Telegram.exe` 放到 `Telegram\` 文件夹

### 2. 首次启动

双击 `启动Telegram.bat`(带日志,方便排查)

首次会弹出黑窗口显示:
```
[步骤 1/2] 正在启动代理...
[信息] 代理已就绪 (耗时 2 秒)
[步骤 2/2] 正在启动 Telegram...
[完成] 已启动。
```

### 3. 在 Telegram 里配置代理(仅首次)

Telegram 启动后:
1. 设置 → 高级 → 连接类型 → 使用自定义代理
2. 添加代理 → SOCKS5
3. 填写:
   - 服务器: `127.0.0.1`
   - 端口: `1080`
   - 用户名/密码: 留空
4. 保存,看到绿点即成功

配置一次后永久保存,以后启动自动生效。

## 日常使用

双击 `启动Telegram.vbs` —— 无黑窗、无弹窗,和启动普通软件一样。

- 代理在后台静默启动
- Telegram 自动使用已保存的代理
- 关闭 Telegram 后,代理自动退出

## 可选:开机自启

把 `启动Telegram.vbs` 的快捷方式放到启动文件夹:

1. 右键 `启动Telegram.vbs` → 创建快捷方式
2. Win+R 输入 `shell:startup` 回车
3. 把快捷方式拖进去

开机后自动启动,无感使用。

## 常见问题

### Q: 启动后 Telegram 连不上?

1. 先用 `启动Telegram.bat` 看日志
2. 检查代理是否就绪(看"代理已就绪"字样)
3. 检查 Telegram 代理设置是否为 127.0.0.1:1080
4. 尝试在 wsproxy 目录下手动运行 `TgWsProxy.exe` 看报错

### Q: 360/火绒拦截?

tg-ws-proxy 开了本地监听端口,可能被误报。添加信任即可。
本方案不修改任何系统设置,纯绿色。

### Q: 端口 1080 被占用?

编辑 `启动Telegram.bat`,把 `PROXY_PORT=1080` 改成其他端口,
同时改 Telegram 代理设置里的端口。

### Q: 可以放 U 盘里用吗?

可以。整个目录绿色无依赖,放哪都能跑。
tg-ws-proxy v1.7.3 起支持便携模式:在 wsproxy 目录下建 `TgWsProxy_data` 文件夹,
配置就存在程序旁边,不写注册表。

## 文件说明

| 文件 | 作用 |
|---|---|
| 启动Telegram.vbs | 静默启动入口(推荐日常用) |
| 启动Telegram.bat | 带日志启动入口(调试用) |

两个文件本质相同,VBS 只是隐藏了 BAT 的黑窗口。
